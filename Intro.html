Step 1: Decide if you want to learn

Do you want to cry in the shower every day for a week because the code that you spent too much time and effort on won't compile just to figure out you forgot a semi-colon? No, you don’t, but that’s coding.

Coding is a massive time investment, there are multiple core languages to learn, and probably hundreds of thousands of frameworks (for each language) that are intended for certain use cases.

Needless to say, in the world of coding, it’s impossible to stop learning. If this isn’t a career you want to pursue in the future, it probably isn't worth the time investment.

Step 2: CS basics

Okay, you've made a bad decision and decided to keep reading, now what? Well, it may be helpful to learn a few universal basics.

There are two types of languages (there are more but these are the main ones), scripting, and design (these aren't official categories but there aren't official categories so I'm going with it).

All scripting languages (Javascript, Python, C#, etc.) use variables, they store different data to be used later in the function, or in other functions depending on the type of variable.

Javascript Examples:

var Example = "this is a variable!";

var ExampleClient = new HTMLElement;

Scripting languages are also mostly based on functions, one function will be the main (which runs when the program runs) function and other functions are called from other functions, and so forth.

However not all languages use the main function, for example, Javascript and Python run code that is just in the text document, and allow you to call functions from there.

There are probably hundreds of other basics I could talk about, but I’m not, so I recommend Googling computer science basics or buying a book such as A Programmer's Guide to Computer Science: A virtual degree for the self-taught developer which is available on Amazon.

Visual languages like HTML (hypertext markup language) are what define the UI (user interface) looks like.

<h1>This is HTML!</h1>

HTML uses elements like divisions (divs), bodies, and spans to contain elements like text, or input! Each element can have properties that allow you to do anything from styling to calling functions in your scripting!

<div name="Container">
   <span class="Question"><strong>Do you enjoy coding?</strong></span>
   <div style="display:inline-block;"><button onclick="this.disabled = true">yes</button><button onclick="this.disabled = true">no</button></div>
</div>

However, just using HTML will make your UI very basic.

CSS is the language of STYLE (it’s not that exciting), you use CSS to make your basic HTML UI look good. It allows you to change colors, change fonts, add drop shadows, add animation, etc.

.Question{
   font-size: 8px;

}

Step 2: Syntax

According to the Oxford Dictionary, the definition of syntax is, “the arrangement of words and phrases to create well-formed sentences in a language.” However, in my opinion, syntax (errors) is why my code doesn't compile even though it’s correct.

Very simply this is how to format English into a word smoothie that a compiler can turn into unreadable system code that the computer can understand.

For example, instead of saying, “I will start watching to see if a button is clicked.” In Javascript I would write:

document.documentElement.addEventListener("mouseup", exampleFunction, false);

In this example, we add an event listener to the element stored in the variable documentElement. This listener is waiting for, “mouseup” over the element when it detects that it will execute the function, “exampleFunction” which is defined somewhere else in the code.

The difference between most scripting languages (apart from certain core functionalities), is the syntax. A function can be defined very differently in C# than it is in Javascript.

Javascript:

function Example(inputVar) { return something; }

C#:

public/private (static?) return-type Example(inputType inputVar) {  return something; }

After learning a language, syntax becomes like a second language. Given that there are way more than those two languages, syntax is most of what programmers will spend their time learning.

Step 3: Your first language

Deciding which language to learn first is hard, but here are some guidelines:

If you want to learn website development, you will have to learn HTML and Javascript to make your first simple website, then you’ll need to learn CSS to make it look good.

If you want to learn app development, learn Kotlin (for Android), Swift (for Apple), and Javascript (for good measure). Both Xcode (Apple) and Android Studio have great resources to help you learn.

If you want to learn game development, learn C#, or start with Gamemaker Studio 2 and learn its native language, GML, it will help you understand the basics.

If you want to learn data processing, learn Python (easy), or C# .NET ASP framework (hard) depending on how hard you want your first experience to be. In my opinion, learning C# first teaches more advanced concepts than Python.

If you’re simply looking to learn to code, and you don't know what you want to do, I recommend data processing, because it will help you understand scripting very well.

I made a questionnaire on CodePen to help you figure out what to learn.

I started with C# because of all you can do with Microsoft’s .NET ASP, such as Discord Bots, Game Development, Console Apps, etc.

I recommend checking out, tools like CodeCademy for studying.

Also, check out the language's documentation. To find documentation, search the name of the language.

Don't go to youtube and search, “how to code” that never works.

Step 4: IDEs and Frameworks

An IDE or Integrated Development Environment is where code is written, it is Microsoft Word but for code.

Some IDEs are online, but they’re slow and filled with ads. These are the two most popular choices:

Microsoft Visual Studio. This IDE is great for those learning C# because of its NuGet Package features, it also has great code suggestions, and integration with programs such as Unity.

Microsoft Visual Studio Code (yes, these two are different). This IDE has great code suggestions, community extensions and tools, and support for almost every language on earth.

Feel free to do some research to pick an IDE.

Step 5: Your first project

After learning the basics of coding, go into the IDE and mess around. I recommend starting small. For beginner game developers, make a simple character controller, or for web developers, make a portfolio or gallery website.

Also, go on sites like CodePen to browse other people's projects and try to understand or even edit their code!

Also sharing starting projects for community feedback is very helpful. Maybe join some Discord servers related to the project and make a GitHub page!

GitHub is also great for storing and sharing code with a team.

Step 6: StackOverflow (and finishing thoughts)

I cannot stress this enough if a program or IDE has built-in shortcuts or functions, use them. They are made by people much smarter than most developers, and it would take anyone years of learning to understand the internal code.

Innovation is not a path best taken alone, pre-made tools are easy to use and speed up the coding process.

Also, even senior developers get stuck and there is no shame in asking for help on Stack Overflow. If someone has had the same problem, learn from their experience.
